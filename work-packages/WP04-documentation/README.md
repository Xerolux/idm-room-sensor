# Documentation

The single source of truth for documentation status is
[`TASKS.md`](../../TASKS.md) (section P4 — Dokumentation und Release) together
with the compact summary in [`PROJECT_STATUS.md`](../../PROJECT_STATUS.md).
Duplicate checklists have been removed to stop them drifting out of sync
(REL-03).

Documentation deliverables and where they live:

- Pages: [`pages/`](../../pages/) (Docsify site)
- Wiki: [`wiki/`](../../wiki/)
- FAQ: [`pages/faq.md`](../../pages/faq.md)
- Troubleshooting: [`pages/troubleshooting.md`](../../pages/troubleshooting.md)
- Ordering: [`docs/ordering/`](../../docs/ordering/)
- Contributing: [`CONTRIBUTING.md`](../../CONTRIBUTING.md)
- Translations: community-driven, tracked via issues

Results, measurements and validation evidence must include conditions,
equipment and raw data.
